## Network Layers

### 1. Compute Layer

Any device can contribute and consume compute:

| Device Type | Contribution | Use Case |
|-------------|--------------|----------|
| Mobile Phone | Light compute, always-on | Relay nodes, light tasks |
| Raspberry Pi | Edge computing | IoT, sensors, distributed tasks |
| Laptop/Desktop | Medium compute | General computing, development |
| GPU Server | Heavy compute | ML training, rendering, scientific |
| Data Center | Massive compute | Large-scale distributed jobs |

**No device too small. No task too big.**

### 2. Storage Layer (CyxCloud)

Decentralized storage that mirrors the compute philosophy:

- **Encrypted shards** distributed across network
- **No single point of failure** or surveillance
- **Redundant storage** across continents
- **Zero-knowledge architecture** - even nodes don't know what they store

## Discussion Topics

### Open Questions

1. **Encryption Strength Model**
   - How exactly does adding nodes strengthen encryption?
   - Is this threshold cryptography? Multi-party computation?

2. **Anonymity vs Performance**
   - More routing = more latency
   - How do we balance privacy and speed?

3. **Bad Actor Prevention**
   - How do we prevent malicious compute?
   - Sandboxing? Verification? Reputation?

4. **Mobile Constraints**
   - Battery drain for always-on nodes
   - Network data costs
   - Background process limitations

5. **Legal Considerations**
   - Anonymous compute can enable illegal activity
   - How do we handle this ethically?

6. **Bootstrapping Problem**
   - Network needs users AND providers
   - How do we solve chicken-and-egg?
